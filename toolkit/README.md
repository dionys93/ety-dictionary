# Etym-Toolkit: Inglisce Dictionary CLI & Translation Engine

A specialized Bash, Node.js, and Python toolkit for managing, auditing, and exporting the **Inglisce Etymological Dictionary**, alongside a deep-learning translation pipeline for full-text books. 

The library is built around a single canonical parser (`etym-parse`) that every other function consumes, ensuring consistent data across all lookup, analysis, and build operations. Presentation is handled entirely by Astro.js; this toolkit's job stops at producing clean, structured JSON and translated text data.

---

## 📁 Repository Structure

* **`data-text/inglisce/dictionary/`** — Core dictionary data organized by alphabetical subdirectories (e.g., `a/animate.txt`).
* **`data-text/books/`** — Raw English source texts (e.g., Sophocles, Shakespeare) waiting to be translated.
* **`data-text/inglisce/books/`** — The final transcribed Inglisce text files.
* **`.cache/ast-books/`** — Intermediate build artifacts (JSON Abstract Syntax Trees) generated by the Python NLP engine.
* **`toolkit/config/`** — Central configuration for paths (`env.sh`) and lookup tables (`languages.tsv`, `parts-of-speech.tsv`).
* **`toolkit/etym-lib.sh`** — The core Bash toolbelt for word lookup, validation, and JSON/CSV compilation.
* **`toolkit/scripts/`** — The Polyglot Build Pipeline:
  * `translate-books.sh` — The master orchestrator for the translation pipeline.
  * `spacy_parser.py` — Python deep-learning NLP parser (spaCy).
  * `transcriber.js` & `utils.js` — Node.js functional transcription and morphological mapping.
  * `build-dictionary.js` — Compiles the dictionary into the master JSON outputs.
* **`toolkit/dist/`** — Automated build output: master JSONL dataset and `translationBrain.json`.
* **`toolkit/tests/`** — Vitest test suite covering the Bash pipeline and translation layer.

## 🚀 Installation & Setup

Because this project utilizes a polyglot architecture (Bash, Node, and Python), you must set up environments for both the frontend API build and the heavy NLP translation engine.

### 1. The Python NLP Environment (spaCy)
The translation engine relies on a Python deep-learning Dependency Parser.
```bash
# Navigate to project root
cd /path/to/project

# Create and activate a virtual environment
python3 -m venv venv
source venv/bin/activate  # (On Windows: venv\Scripts\activate)

# Install requirements and the heavy Transformer model
pip install fastapi uvicorn spacy
python -m spacy download en_core_web_trf
```

### 2. The Node.js Environment
Required for dictionary compilation and applying Inglisce suffixes.
```bash
npm install
```

### 3. The Bash Toolbelt
Source the library from within the `toolkit/` directory to activate all CLI tools in your session:
```bash
source toolkit/etym-lib.sh
```

## 📖 The Translation Pipeline

To translate an entire folder of English books into Inglisce, we use a highly optimized, decoupled 2-step pipeline.

**To run the pipeline:**
```bash
./toolkit/scripts/translate-books.sh
```

### How it works:
1. **Phase 1: Deep Learning (Python):** `spacy_parser.py` reads the English text and uses the `en_core_web_trf` neural network to generate a highly accurate Abstract Syntax Tree (AST). It perfectly identifies Verb phrases, Plural Nouns, and Named Entities, saving them to `.cache/ast-books/`.
2. **Phase 2: Transcription (Node.js):** `transcriber.js` consumes the cached AST JSON. It acts as a pure, functional mapper—looking up root words in `translationBrain.json` and applying the correct morphological suffixes (via `utils.js`) without freezing the main thread. It outputs the final `.txt` files to `data-text/inglisce/books/`.
3. **Missing Words Tracker:** The script will automatically aggregate a globally sorted list of missing dictionary words in your terminal output so you know exactly what words to add to your dictionary next.

## 🛠️ CLI Toolkit Commands

### `etym-lint [path]`
Scans the specified dictionary directory (defaults to all) for formatting errors in the raw `.txt` files.

* `[FATAL]` — file is empty
* `[ERROR]` — missing or malformed POS tag `()`, missing language tag `[]`, or both tags on the same line
* `[WARN]` — trailing whitespace, or one or more stanzas with no resolvable language origin

The lint report also includes a **verb conjugation coverage** summary: how many verb stanzas use standard `-s -d/-ed -ing` vs non-standard forms, with a full listing of non-standard entries for auditing.

Returns exit code 1 if any fatal errors or standard errors are found.

### `etym-trim [path]`
Strips trailing whitespace from all `.txt` files in a directory. Resolves all trailing whitespace lint warnings in one pass. macOS and Linux compatible.

### `etym-create-histories [-d] [-v] [--dirs <a,b,c>]`
Splits each multi-stanza `.txt` file into individual per-definition history files, one file per stanza, named `<word>_<pos>.txt`.

* `-d, --dry-run` — preview without writing files
* `-v, --verbose` — show each extracted file path
* `--dirs <a,b,c>` — process only specific letter directories

## 🕸 Graphing & Visualization

### `etym-graph [path] [-o <file>]`
Builds a `{nodes, edges}` JSON graph of all etymological relationships across the dictionary, suitable for force-directed graph renderers. Outputs to `etym_graph.json` by default.

> **Note:** requires jq 1.6+.

### `etym-visualize [graph_file]`
Converts `etym_graph.json` into a `.md` or web-friendly format consumable by modern network visualization tools.
